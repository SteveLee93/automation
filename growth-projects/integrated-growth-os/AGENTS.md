# AGENTS.md

## 프로젝트 목적
이 프로젝트는 X, 뉴스, YouTube라는 서로 다른 소스의 수집 결과를 연결해
오늘 반복적으로 등장한 핵심 흐름을 통합 cluster 문서와 통합 분석 문서로 정리하는 것을 목표로 한다.

## 먼저 참고할 공통 문서
- `../shared/AGENTS.md`
- `../shared/config/categories.md`
- `../shared/config/scoring_rules.md`
- `../shared/docs/output_rules.md`

## 통합 프로젝트 역할
- 18:00~06:00에는 소스별 raw 수집 상태를 묶는 시스템용 log를 만든다.
- X의 빠른 신호는 초기 감지로 본다.
- 뉴스의 기사 원문과 주요 매체 보도는 사실 확인과 맥락 보강으로 본다.
- YouTube의 긴 설명형 콘텐츠는 해설과 학습 가치 보강으로 본다.
- 깊은 해석보다 묶기, 교차 확인, 우선순위화에 집중한다.

## 통합 원칙
- 같은 단어가 아니라 같은 사건, 같은 논점, 같은 흐름인지 판단해서 묶는다.
- X와 뉴스가 모두 존재하면 `Cross-confirmed`로 표시한다.
- X만 있으면 `X-only signal`, 뉴스만 있으면 `News-only signal`로 표시한다.
- YouTube만 있으면 `YouTube-only signal`로 표시한다.
- 세 소스 중 둘 이상이 정렬되면 `Cross-confirmed`로 본다.
- X의 해석이 강하고 뉴스 근거가 없으면 중요도를 과도하게 높이지 않는다.
- 뉴스 기사 원문이 강하고 X 반응이 적더라도 오늘 흐름상 중요하면 cluster에 포함한다.
- YouTube 설명이 좋아도 최신성과 사실 확인이 약하면 과도하게 높이지 않는다.
- 같은 사실의 반복 전달은 줄이되, 서로 다른 지역 관점이나 시장 반응은 보조 메모로 남긴다.

## 출력 원칙
- 모든 주요 cluster에는 제목, 핵심 주제, 유형, 카테고리, 중요도, 대표 항목, 저녁 재검토 이유가 있어야 한다.
- 대표 항목은 가능하면 뉴스 1개, X 1개, YouTube 1개 관점으로 균형 있게 고른다.
- raw log 문서는 상태 확인과 누락 진단 중심으로 짧게 쓰고, 깊은 분석 문장을 넣지 않는다.
- 문서 마지막에는 `Noon Notes`와 `Cluster Diagnostics`를 반드시 포함한다.
