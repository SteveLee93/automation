# project_principles.md

## 소스 분리 원칙
- X, 뉴스, YouTube, 시장 데이터 같은 소스는 프로젝트를 분리한다.
- 수집 방식, 신뢰도 판단, 중복 제거 방식이 다르면 별도 프로젝트로 본다.

## 역할 분리 원칙
- 회사 시간에는 수집과 정리를 우선한다.
- 집에서는 해석, 행동 제안, 기록 자산화를 우선한다.

## 유지보수 원칙
- 공통 카테고리와 출력 규칙은 `shared/`에서 관리한다.
- 소스 특화 규칙은 각 프로젝트 안에서만 관리한다.
- 후보 소스는 운영 데이터와 분리해 `data/candidates/`에 축적한다.

## 확장 원칙
- 새 소스를 추가할 때는 기존 프로젝트에 억지로 넣지 말고 새 프로젝트를 만든다.
- 통합 분석은 소스별 프로젝트가 안정화된 뒤 별도 프로젝트로 붙인다.
