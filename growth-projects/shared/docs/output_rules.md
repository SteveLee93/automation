# output_rules.md

## 공통 출력 원칙
- 모든 산출물은 Markdown으로 저장한다.
- 문서 맨 위에는 실행 시각과 입력 범위를 남긴다.
- 주요 항목마다 출처, 시각, 카테고리, 요약을 포함한다.
- 마지막에는 반드시 요약 섹션을 둔다.

## 수집 문서 규칙
- 읽기용 섹션과 진단용 섹션을 함께 둔다.
- Diagnostics에는 개수, 분포, 제외 이유, 중복 제거 정보를 포함한다.
- 최신성 판단이 중요한 소스는 시간 범위를 명시한다.

## 분석 문서 규칙
- 흐름 요약, 중요성, 나와의 관련성, 다음 행동을 분리한다.
- 막연한 감상보다 실행 가능한 문장을 우선한다.

## handoff 문서 규칙
- 집에서 10~15분 안에 훑어볼 수 있게 압축한다.
- 너무 많은 항목보다 대표성 있는 항목을 남긴다.
